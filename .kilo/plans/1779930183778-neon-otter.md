# Offset Learning & Sleep Mode Implementation Plan

## Overview
Add two major features to the multizone mini-split thermostat integration:
1. **Offset Learning System** - ML-based regression that learns the temperature offset between zone thermostats and mini-splits, using outside temperature as a predictor
2. **Per-Zone Sleep Mode** - Prevents continuous beeping adjustments in zones (e.g., bedrooms) during sleep by forcing a preset, while still allowing mode switching

## Architecture

### Offset Learning
- **Input**: Outside temperature (global entity) + zone current temp + mini-split current temp
- **Model**: Simple linear regression `offset = a * outside_temp + b`
- **Storage**: Sliding window of data points (30 days), persisted via Home Assistant storage
- **Application**: Learned offset adjusts the temperature sent to the mini-split: `mini_split_target = thermostat_target + offset`
- **Update cadence**: Recalculate periodically; only push new offset if debounce criteria met

### Sleep Mode (Per-Zone)
- Each zone can configure a sleep mode entity (e.g., `input_boolean.bedroom_sleep`)
- When entity state is "on": force zone to configured sleep preset
- **Does NOT prevent mode switching** - HVAC mode changes still propagate
- **Prevents temperature adjustments** - no beeping from mini-split receiving new setpoints
- Each zone configures its own sleep preset (e.g., "eco" for sleeping)

### Debounce System
- **Time-based**: Minimum interval between temperature adjustments per zone (default: 15 minutes)
- **Threshold-based**: Only adjust when new offset differs from current by ≥ threshold (default: 0.5°F)
- **Both must be satisfied** to trigger an adjustment

### Step 2: Create Offset Learner Module (`offset_learner.py`)

New file implementing simple linear regression with sliding window:

**Key components:**
- `OffsetLearner` class per zone
- `add_data_point(outside_temp, zone_temp, minisplit_temp, timestamp)` - records a data point
- `get_predicted_offset(outside_temp) -> float` - returns predicted offset using current model
- `_fit_model()` - recalculates regression coefficients from stored data
- `_prune_old_data()` - removes data points outside the learning window
- `_persist()` / `_load()` - saves/loads data via HA storage helper

**Regression math (no external dependencies):**
```python
# Simple linear regression: y = a*x + b
# where y = offset (minisplit_temp - zone_temp), x = outside_temp
# a = (n*Σxy - Σx*Σy) / (n*Σx² - (Σx)²)
# b = (Σy - a*Σx) / n
```

**New methods:**

- Data collection happens on every zone state change
- Offset recalculation happens on a timer + outside temp changes

## Key Design Decisions

1. **No external ML dependencies** - Simple linear regression implemented manually to avoid requiring sklearn/numpy in HA
2. **Per-zone sleep mode** - More flexible than global; some zones are bedrooms, others aren't
3. **Global outside temp** - Single source of truth for the whole system
4. **Offset applied to mini-split target** - Adjusts what we send to the mini-split, not how we read the thermostat
5. **Both debounce criteria required** - Prevents both rapid adjustments AND tiny unnecessary changes
6. **Mode switching still active during sleep** - Important for days with dramatic outside temperature changes
7. **Data persisted via HA storage** - Survives restarts; uses `storage` helper pattern

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Not enough data for regression initially | Start with offset=0, gradually learn; show sample count in sensor |
| Outside temp entity unavailable | Fall back to no offset adjustment; log warning |
| Sleep mode entity flickers | Only evaluate sleep state when determining target, not continuously |
| Regression produces extreme offsets | Clamp offset to reasonable range (e.g., ±5°F) |
| Storage grows unbounded | Sliding window pruning on every data point addition |
